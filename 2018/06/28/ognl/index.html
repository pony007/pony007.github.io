OGNL

发表于 2018-06-28 | 分类于 struts

表达式: 主要用于计算或取值

struts表达式

  1. Struts2支持多种表达式语言,比如OGNL,JSTL,Groovy,Velocity
  2. OGNL可以存取对象的任意属性,调用对象的方法,遍历整个对象的结构图,实现字段类型转化等功能.因此struts2将OGNL表达式作为默认表达式
  3. 思考:
    • 中级的JSP中含有EL表达式: EL表达式只能读取容器或URL中的数据
    • struts2的视图页面: freemarker,jsp,html等等.常用JSP,在struts2的JSP中含有2种表达式: EL表达式,OGNL表达式

ognl实现:

  1. ValueStack[实现类为OgnlValueStack]是OGNL的根对象,该对象提供了OGNL整体的容器对象和默认数据
  2. ActionContext提供了获得其他OGNL对象的方式,因此ActionContext称为ONGL的上下文对象[提供整体环境]

ValueStack对象

  1. ValueStack[实现类为OgnlValueStack]是OGNL的根对象,该对象提供了OGNL整体的容器对象和默认数据
  2. 通常通过操作ValueStack访问对象栈[root或_root],格式: 名称 [复合名称]
  3. 当操作ValueStack对象栈时,默认先从栈顶对象搜索,找数据即返回. 若找不到,将搜索对象栈中下1个对象中的相关属性
  4. 当Struts2接受一个请求后,迅速创建ActionContext,ValueStack,action.然后把action存放进ValueStack,所以可以使用OGNL访问action的实例变量
  5. Struts2封装了request对象的getAttribute(),当request容器找不到相关数据时,会到ValueStack中搜索相关数据.

注意：ValueStack的有效范围只在同1个请求中.

  1. 延伸: 数据回显
    • 数据回显: 利用属性注入与转发

注意：

- OGNL表达式不能单独编写,必须与struts2的标签配置使用.
- EL表达式可以编写页面的几乎所有位置[文本,标签属性值或部分],Java片段除外.

ActionContext对象

  1. ActionContext提供了获得其他OGNL对象的方式,因此ActionContext称为ONGL的上下文对象[提供整体环境]
  2. 通常操作ActionContext访问Map栈中的数据,格式: #名称
  3. #相当于ActionContext,整体表示取出指定键名的值.比如 #session表示取出session对象,#myUser表示取出键名为myUser的值
  4. “#名称”表示访问ActionContext[Map栈]中的数据.”名称”,表示访问ValueStack的对象栈[root]

OGNL常量: [数据类型]

  1. 字符串常量: “字符串” ‘字符串’ [若字符串中字符个数为1个,必须使用双引号]
  2. 字符常量: ‘符号’
  3. 数值常量: 数字
  4. 布尔常量: true false

OGNL操作符: [运算符]

  1. OGNL支持Java的所有操作符(+,-,*/等等),并提供了一些特有的操作符
  2. OGNL提供的特殊操作符: “,”,”{}”,”in”,”not in”等等

OGNL表达式符号

  1. #
  2. 访问OGNLContext属性,比如:#session,#userList
  3. 创建Map,比如: #{0:’男’,1:’女’}
  4. 用于过滤/投影集合,比如: persons.{?#this.age>30}
  5. % 用于解决不支持OGNL表达式的struts2标签
  6. .$ 通常在struts.xml中使用,生成动态的结果或参数

注意：因为EL表达式已经使用$,struts2为了防止冲突,选用#.

OGNL与对象:

  1. 访问对象栈中对象的属性:<s:property value=”属性名” />
  2. 访问对象栈中对象的方法:<s:property value=”方法名()” />
  3. 访问Map栈中对象的属性:<s:property value=”#对象.属性名” />
  4. 访问Map栈中对象的方法:<s:property value=”#对象.方法名()” />
  5. 访问静态属性:<s:property value=”@类全名@属性名” />
  6. 访问静态方法:<s:property value=”@类全名@方法名()” />
  7. 访问构造方法:<s:property value=”new 类全名()” />

OGNL与集合:

OGNL生成List对象: {元素1,元素2,元素3…}

OGNL生成Map对象: #{键名:值,键名:值,…}

操作：

  • OGNL访问List中的某个元素:<s:property value=”list对象[索引]” />
  • OGNL访问Map中的某个元素:<s:property value=”map对象.键名” />
  • OGNL访问Map中的某个元素:<s:property value=”map对象[‘键名’]” />
  • OGNL访问Map的所有Key/Value:<s:property value=”map对象.keys/values” />
  • OGNL访问容器的大小:<s:property value=”容器对象.size()” />
